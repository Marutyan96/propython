принципы ООП

Наследование, инкапсуляция, полиморфизм и абстракция — это ключевые концепции объектно-ориентированного программирования, которые позволяют создавать более сложные структуры классов и повторно использовать код. Это упрощает разработку и поддержку программного обеспечения.

Наследование

Наследование — это один из основных принципов объектно-ориентированного программирования, который позволяет создавать новый класс на основе существующего. Новый класс наследует свойства и методы родительского класса.

Создание базового класса

Сначала создадим базовый класс Vehicle, который будет содержать общие методы и свойства, такие как марка, модель и год выпуска.


class Vehicle:
    def __init__(self, make, model, year):
        self.make = make
        self.model = model
        self.year = year
    
    def info(self):
        f"{self.year} {self.make} {self.model}"
    
    def start_engine(self):
        "Двигатель запущен!"

Создание дочерних классов

Теперь создадим классы Car и Truck, которые наследуют от базового класса Vehicle.


class Car(Vehicle):
    def start_engine(self):
        "Двигатель автомобиля запущен!"

class Truck(Vehicle):
    def start_engine(self):
        "Двигатель грузовика запущен!"

Теперь мы можем создавать экземпляры дочерних классов и использовать их методы.


car = Car("Toyota", "Camry", 2020)
print(car.info())  # Вывод: 2020 Toyota Camry
print(car.start_engine())  # Вывод: Двигатель автомобиля запущен!

truck = Truck("Ford", "F-150", 2019)
print(truck.info())  # Вывод: 2019 Ford F-150
print(truck.start_engine())  # Вывод: Двигатель грузовика запущен!

Переопределение методов

Дочерние классы могут переопределять методы родительского класса, чтобы изменить их поведение. Например, мы можем создать класс ElectricCar, который также наследует от Car.


class ElectricCar(Car):
    def start_engine(self):
        "Электродвигатель запущен!"

electric_car = ElectricCar("Tesla", "Model 3", 2021)
print(electric_car.info())  # Вывод: 2021 Tesla Model 3
print(electric_car.start_engine())  # Вывод: Электродвигатель запущен!

Инкапсуляция

Инкапсуляция — это принцип, который позволяет скрывать внутренние детали реализации класса и предоставлять доступ к ним только через методы. Это позволяет защитить данные и предотвратить их некорректное использование.

В Python инкапсуляция достигается с помощью использования приватных атрибутов, которые начинаются с одного или двух подчеркиваний.


class Person:
    def __init__(self, name):
        self.__name = name  # Приватный атрибут
    
    def get_name(self):
        return self.__name

person = Person("Иван")
print(person.get_name())  # Вывод: Иван

Полиморфизм

Полиморфизм позволяет использовать одинаковые интерфейсы для различных типов объектов. Это означает, что один и тот же метод может вести себя по-разному в зависимости от объекта, который его вызывает.

В нашем примере мы можем создать список транспортных средств и вызвать метод start_engine для каждого из них:


vehicles = [Car("Toyota", "Camry", 2020), 
              Truck("Ford", "F-150", 2019), 
              ElectricCar("Tesla", "Model 3", 2021)]

for vehicle in vehicles:
    print(vehicle.start_engine())  # Вывод: Двигатель автомобиля запущен!, Двигатель грузовика запущен!, Электродвигатель запущен!

Абстракция

Абстракция — это процесс выделения общих характеристик объектов и игнорирование несущественных деталей. В Python это можно реализовать с помощью абстрактных классов и интерфейсов, которые определяют методы, которые должны быть реализованы в дочерних классах.

Для создания абстрактного класса в Python используется модуль abc (Abstract Base Class).


from abc import ABC, abstractmethod

class Vehicle(ABC):
    @abstractmethod
    def start_engine(self):
        pass

class Car(Vehicle):
    def start_engine(self):
        return "Двигатель автомобиля запущен!"

class Truck(Vehicle):
    def start_engine(self):
        return "Двигатель грузовика запущен!"

class ElectricCar(Vehicle):
    def start_engine(self):
        return "Электродвигатель запущен!"
